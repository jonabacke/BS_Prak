9  * @brief Based on a LINUX message queue (s. mq_overview) this module implements a task
10  * queue of a thread pool. Each queue element stores one task. Two items define a
11  * task
12  * (1) A pointer to the function that implements the behavior of the task
13  * (2) A structure that stores the arguments used by this function
14  *
15  * To support argument structures of different size, a queue element is divided into
16  * two parts. The first part is a header that stores the function pointer and the size
17  * of the structure that contains the arguments of the function. The argument structure
18  * itself is stored as a sequence of chars, which is located in memory behind the header.
19  *
20  * Remarks:
21  * - The maximum size of a queue and the maximum size of a queue element is given
22  * in /proc file system (e.g. /proc/sys/fs/mqueue/msg_max).
23  * - This is the second release of the task queue module. In opposite to the first one,
24  * the argument of a task is stored in the message queue element. The first release
25  * stored such an argument on the heap.
26  *
27  ******************************************************************
28  */
